# 발로란트 방송 오버레이 (VCT 퍼시픽 스타일)

개인 방송인이 OBS에 송출할 수 있는 발로란트 대회용 오버레이입니다.
운영자(옵저버)가 **컨트롤 패널**에서 값을 입력하면, OBS의 **오버레이**에 실시간으로 반영됩니다.

표시 요소: 팀명·팀로고 · 스코어보드 · 맵/라운드 · 공수(ATK/DEF) 표시 · 선수 명단 ·
타이머 · 무기/방어구 상태 · 궁극기 포인트 · 스킬(C/Q/E) 유무 · 선수 캠 슬롯(ON/OFF).

---

## ⚠️ 먼저 알아둘 것 — 데이터는 "수동 입력"입니다

발로란트의 무기·궁극기·체력 같은 **실시간 인게임 데이터를 자동으로 받아오는 공식 API는
라이엇 방송 파트너에게만 제공**됩니다. 개인은 접근할 수 없습니다.

따라서 이 프로그램은 **운영자가 경기를 보면서 컨트롤 패널에서 직접 값을 바꾸는 방식**입니다.
혼자 방송하며 동시에 모든 값을 입력하긴 어려우므로, 보통은 ① 스코어/공수/맵 정도만
간단히 갱신하거나, ② 친구 1명이 옵저버 역할로 패널을 조작하는 식으로 운영합니다.

---

## 설치 & 실행 (3단계)

1. **Node.js 설치** — https://nodejs.org (LTS 버전). 추가 라이브러리 설치 불필요.
2. 이 폴더에서 터미널을 열고:
   ```
   node server.js
   ```
3. 터미널에 표시되는 주소를 사용합니다:
   - **OBS 브라우저 소스**: `http://localhost:8080/overlay`
   - **컨트롤 패널(운영자)**: `http://localhost:8080/control`
   - 폰/다른 PC에서 조작하려면 표시된 `http://192.168.x.x:8080/control` 주소 사용

---

## OBS 설정

1. 소스 → **추가 → 브라우저** 선택.
2. URL에 `http://localhost:8080/overlay` 입력.
3. 너비 **1920**, 높이 **1080** 입력 → 확인.
4. 게임 캡처 소스 **위에** 오버레이가 오도록 순서 정렬.
5. 배경은 자동으로 투명하므로 게임 화면이 비칩니다.

자세한 점검은 `OBS-setup-checklist.md` 참고.

---

## 사용법

컨트롤 패널에서:
- **경기 상태**: 맵, 라운드, 국면(BUY/LIVE/PLANTED/POST), 타이머(▶로 카운트다운), 공수 교체, 스파이크.
- **팀 A/B**: 팀명, 약칭(트라이코드), 컬러, 로고 URL, 스코어(+/−).
- **선수 명단**: 이름, 에이전트, 생존/사망, 무기, HP, 방어구(0~2), 크레딧, 궁극기(현재/필요), 스킬 C/Q/E.
- **캠**: ON/OFF, 좌하단·우하단 위치, 라벨. 캠을 켜면 오버레이에 빈 프레임이 생기며,
  OBS에서 **실제 웹캠/캡처 소스를 그 위치에 겹쳐 배치**하면 됩니다.

값을 바꾸면 즉시 오버레이에 반영됩니다.

## 커스터마이징

- **색·폰트·크기**: `public/overlay.css` 상단의 CSS 변수(`:root`)에서 조정.
- **에이전트/무기/맵 목록**: `public/shared.js`의 `AGENTS`, `WEAPONS`, `MAPS` 배열 수정.
- **팀 로고**: 컨트롤 패널의 "로고" 칸에 이미지 URL을 넣거나, 로컬 파일은 `public/` 폴더에
  넣고 `http://localhost:8080/파일명.png` 형태로 입력.

## 구성 파일

```
valorant-overlay/
├─ server.js               # 무의존성 Node 서버 (정적 제공 + 상태 중계, SSE)
├─ package.json
├─ demo.html               # 단일 파일 미리보기(서버 없이 브라우저로 바로 열기)
├─ README.md
├─ OBS-setup-checklist.md
└─ public/
   ├─ overlay.html         # OBS 브라우저 소스
   ├─ overlay.css          # 디자인 토큰 + 컴포넌트
   ├─ overlay.js           # 렌더링 + 실시간 수신(SSE)
   ├─ control.html         # 운영자 컨트롤 패널
   ├─ control.js           # 패널 로직(상태 전송)
   └─ shared.js            # 기본 상태/목록 데이터
```

## 동작 원리

`control.js`가 변경된 전체 상태를 `POST /update`로 서버에 보내고,
서버는 그 상태를 **SSE(`/events`)** 로 연결된 모든 오버레이에 즉시 전파합니다.
별도 데이터베이스나 외부 서비스 없이 로컬에서 완결됩니다.

> 팁: 미리보기만 빠르게 보고 싶으면 `demo.html`을 브라우저로 그냥 더블클릭하세요.
> (서버 없이 컨트롤이 내장된 데모 버전)
